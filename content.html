Lists and dictionaries can be combined in powerful ways: you can store lists as dictionary values, or nest dictionaries inside other dictionaries. This creates flexible data structures for organizing complex information.

Lists as Dictionary Values

Since dictionary values can be any type, you can store lists as values:

student = {"name": "Alice", "grades": [90, 85, 92]} print(student) student["subjects"] = ["Math", "English", "Science"] print(student)

Each value is a list, so you can access individual items using additional indexing:

student = {"name": "Alice", "grades": [90, 85, 92]} print(student["grades"]) # Get the list print(student["grades"][0]) # Get first grade print(student["grades"][-1]) # Get last grade

You can also modify the list values using list methods:

student = {"name": "Alice", "grades": [90, 85]} student["grades"].append(92) print(student)

Nested Dictionaries

Dictionaries can also contain other dictionaries as values, creating nested structures:

person = {"name": "Alice"} person["address"] = {"city": "New York", "zip": "10001"} print(person) print(person["address"]["city"])

To access a value in a nested dictionary, use chained bracket notation: person["address"]["city"]. This reads from the outside in: get the "address" value from person, then get the "city" value from that dictionary.

Practical Example: Building More Complex Data

Nested collections are useful for organizing real-world data. Here's an example of a book database:

library = { "book1": { "title": "Python Basics", "author": "Jane Smith", "pages": 350, "tags": ["programming", "python", "beginner"] }, "book2": { "title": "Web Development", "author": "John Doe", "pages": 420, "tags": ["web", "javascript", "html"] } } print(library["book1"]["title"]) print(library["book1"]["tags"][0])

This structure combines dictionaries (for labeled data) and lists (for ordered collections) to organize information hierarchically.
